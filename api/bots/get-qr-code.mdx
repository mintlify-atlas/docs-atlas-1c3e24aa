## Method Signature

```typescript
getBotQrCode(builderBotId: string): Promise<{ qr: string }>
```

Retrieves a QR code for authenticating a bot with WhatsApp. The user scans this QR code with their WhatsApp mobile app to link the bot to their WhatsApp account.

## Parameters

<ParamField path="builderBotId" type="string" required>
  The BuilderBot instance ID (not the GestorPro bot ID). This is the `id_builderBot` field from the Bot object.
</ParamField>

<Note>
This method uses the `id_builderBot` field, not the GestorPro `id`. You can find this value in the Bot object's `id_builderBot` property.
</Note>

## Returns

<ResponseField name="Promise<{ qr: string }>" type="Promise">
  Returns a promise that resolves to an object containing the QR code
  
  <Expandable title="Response Object">
    <ResponseField name="qr" type="string" required>
      Base64-encoded QR code image or QR code data string. This can be displayed as an image or rendered using a QR code library.
    </ResponseField>
  </Expandable>
</ResponseField>

```typescript Auto-Refresh QR Code
import { getBotQrCode } from './services/api';

async function autoRefreshQrCode(
  builderBotId: string,
  onUpdate: (qr: string) => void
) {
  const REFRESH_INTERVAL = 60000; // 1 minute
  
  async function fetchAndUpdate() {
    try {
      const { qr } = await getBotQrCode(builderBotId);
      onUpdate(qr);
      console.log('QR code refreshed');
    } catch (error) {
      console.error('Failed to refresh QR code:', error);
    }
  }
  
  // Initial fetch
  await fetchAndUpdate();
  
  // Refresh periodically
  const intervalId = setInterval(fetchAndUpdate, REFRESH_INTERVAL);
  
  // Return cleanup function
  return () => clearInterval(intervalId);
}

// Usage
```

## Error Handling

<CodeGroup>
```typescript Try-Catch Pattern
import { getBotQrCode } from './services/api';

async function safeGetQrCode(builderBotId: string) {
  try {
    const { qr } = await getBotQrCode(builderBotId);
    return qr;
  } catch (error) {
    if (error instanceof Error) {
      if (error.message.includes('not found')) {
        console.error('BuilderBot instance not found');
        throw new Error('Bot configuration error. Please check BuilderBot ID.');
      } else if (error.message.includes('Failed to fetch')) {
        console.error('Network error');
        throw new Error('Cannot connect to server. Check your internet connection.');
      } else {
        console.error('Unknown error:', error.message);
        throw new Error('Failed to generate QR code. Please try again.');
      }
    }
    throw error;
  }
}
```

```typescript With Retry
import { getBotQrCode } from './services/api';

async function getQrCodeWithRetry(builderBotId: string, maxAttempts = 3) {
  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      const { qr } = await getBotQrCode(builderBotId);
      return qr;
    } catch (error) {
      console.error(`Attempt ${attempt} failed:`, error);
      
      if (attempt === maxAttempts) {
        throw new Error(`Failed to get QR code after ${maxAttempts} attempts`);
      }
      
      // Wait before retrying
      await new Promise(resolve => setTimeout(resolve, 2000 * attempt));
    }
  }
  
  throw new Error('Unexpected error in retry logic');
}
```

## Important Notes

<Warning>
QR codes expire after a certain period (typically 60 seconds). Implement auto-refresh functionality to generate new QR codes if the user takes too long to scan.
</Warning>

<Note>
The `builderBotId` parameter is the BuilderBot instance ID (`id_builderBot`), not the GestorPro bot ID (`id`). Make sure to use the correct ID when calling this method.
</Note>

<Note>
After the user scans the QR code, you still need to call `connectBot()` to activate the bot and change its status to "online".
</Note>

## Best Practices

1. **Display clear instructions** alongside the QR code
2. **Implement QR code refresh** - codes expire quickly
3. **Show loading states** while fetching the QR code
4. **Handle errors gracefully** with user-friendly messages
5. **Confirm scan completion** before proceeding to connect
6. **Use appropriate QR size** - minimum 300x300px for easy scanning
7. **Test on mobile devices** to ensure scanability
