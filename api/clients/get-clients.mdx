## Method Signature

```typescript
export const getClients = async (): Promise<Client[]>
```

<Note>
  This method is defined in `~/workspace/source/services/api.ts:99`
</Note>

## Description

Retrieves all client records associated with the authenticated user's spreadsheet. This method does not require any parameters and returns a complete list of clients.

## Parameters

This method does not accept any parameters.

## Return Type

Returns a Promise that resolves to an array of `Client` objects.

```typescript React Component
import { useEffect, useState } from 'react';
import { getClients } from './services/api';
import type { Client } from './types';

function ClientList() {
  const [clients, setClients] = useState<Client[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const loadClients = async () => {
      try {
        const data = await getClients();
        setClients(data);
      } catch (err) {
        setError(err instanceof Error ? err.message : 'Failed to load clients');
      } finally {
        setLoading(false);
      }
    };

    loadClients();
  }, []);

  if (loading) return <div>Loading clients...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {clients.map(client => (
        <li key={client.id}>
          {client.name} - {client.contactPerson}
        </li>
      ))}
    </ul>
  );
}
```

```typescript With Error Handling
import { getClients } from './services/api';
import type { Client } from './types';

const loadClientsWithRetry = async (maxRetries = 3): Promise<Client[]> => {
  let lastError: Error | null = null;
  
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      const clients = await getClients();
      return clients;
    } catch (error) {
      lastError = error instanceof Error ? error : new Error('Unknown error');
      console.warn(`Attempt ${attempt} failed:`, lastError.message);
      
      if (attempt < maxRetries) {
        // Wait before retrying (exponential backoff)
        await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
      }
    }
  }
  
  throw lastError || new Error('Failed to fetch clients after retries');
};
```

## Error Handling

The method may throw errors in the following scenarios:

<Warning>
  Always wrap `getClients()` calls in try-catch blocks to handle potential errors gracefully.
</Warning>

### Common Errors

<AccordionGroup>
  <Accordion title="User is not authenticated. No session found.">
    **Cause**: No valid session found in localStorage
    
    **Solution**: Ensure the user is logged in and the session is properly stored
    
    ```typescript
    // Check session before calling
    const session = localStorage.getItem('user_session');
    if (!session) {
      // Redirect to login
      window.location.href = '/login';
    }
    ```
  </Accordion>
  
  <Accordion title="ID de hoja de cálculo no encontrado en la sesión">
    **Cause**: Session exists but doesn't contain `id_sheet` property
    
    **Solution**: Ensure the login process correctly stores the spreadsheet ID
  </Accordion>
  
  <Accordion title="Failed to fetch">
    **Cause**: Cannot connect to the Google Apps Script backend
    
    **Solution**: 
    - Verify the `DATA_SCRIPT_URL` is correct
    - Check Google Apps Script deployment settings
    - Ensure "Who has access" is set to "Anyone"
  </Accordion>
  
  <Accordion title="La respuesta del backend no es un JSON válido">
    **Cause**: Backend returned non-JSON response
    
    **Solution**: Check Google Apps Script logs for errors in the backend code
  </Accordion>
</AccordionGroup>

## Implementation Details

The method internally calls:

```typescript
return callAppsScript('getClients');
```

This sends a POST request to the Google Apps Script backend with:
- Action: `'getClients'`
- Payload: User's spreadsheet ID from session
- No additional data required
