## Project Properties

### Core Fields

Each project includes the following information:

```typescript
interface Project {
  id: string;               // Unique identifier
  name: string;             // Project name
  clientId: string;         // Associated client
  clientName?: string;      // Client display name
  startDate: string;        // Project start date
  endDate: string;          // Project end date
  budget: number;           // Project budget
  status: ProjectStatus;    // Current status
}
```

### Project Status

Projects can be in one of four states:

```typescript
enum ProjectStatus {
  NotStarted = 'No Iniciado',
  InProgress = 'En Progreso',
  Completed = 'Completado',
  OnHold = 'En Pausa',
}
```

<Info>
  Project status helps you track progress at a glance and filter projects by their current state.
</Info>

## Managing Projects

<Tabs>
  <Tab title="Create Project">
    ### Creating a New Project
    
    <Steps>
      <Step title="Open Project Form">
        Click the **"Nuevo Proyecto"** button in the Projects header.
      </Step>
      
      <Step title="Fill in Details">
        Complete the project form with:
        - Project name
        - Client selection (dropdown)
        - Start and end dates
        - Budget amount
        - Initial status
      </Step>
      
      <Step title="Save">
        Click save to create the project. It will appear in the project grid sorted by start date.
      </Step>
    </Steps>
    
    <Note>
      Projects are automatically sorted by start date (newest first) after creation.
    </Note>
  </Tab>
  
  <Tab title="Edit Project">
    ### Editing an Existing Project
    
    You can modify any project details:
    
    1. Click the **Edit** button on a project card
    2. Modify any fields in the project form
    3. Save changes to update the project
    
    <Warning>
      Changing a project's client will update the association but won't affect historical data.
    </Warning>
  </Tab>
  
  <Tab title="Delete Project">
    ### Deleting a Project
    
    <Warning>
      Deleting a project will **permanently remove** all associated data:
      - Notes
      - Links
      - Prompts
      
      This action cannot be undone.
    </Warning>
    
    To delete a project:
    
    1. Click the **Delete** button on a project card
    2. Confirm the deletion in the dialog
    3. The project and all associated data will be removed
  </Tab>
</Tabs>

### Grid Layout

Projects are displayed in a responsive grid:

## Project Operations

The Projects component handles these operations:

```typescript
// Fetch projects and clients
const [projectsData, clientsData] = await Promise.all([
  getProjects(),
  getClients()
]);

// Create new project
await addProject(projectData);

// Update existing project
await updateProject({ ...editingProject, ...projectData });

// Delete project
await deleteProject(projectId);
```

<Note>
  Projects and clients are fetched in parallel for optimal performance.
</Note>

## Associated Data

Projects can have three types of associated data:

<CardGroup cols={3}>
  <Card title="Notes" icon="file-lines">
    Free-form text notes for project documentation, meeting notes, or reminders.
  </Card>
  
  <Card title="Links" icon="link">
    URLs to important resources like documentation, designs, or staging environments.
  </Card>
  
  <Card title="Prompts" icon="comment-dots">
    Stored AI prompts and responses for reference and reuse.
  </Card>
</CardGroup>

<Warning>
  All associated data is **permanently deleted** when a project is removed.
</Warning>

## Loading States

The Projects module provides visual feedback during operations:

- **Initial load**: Centered spinner while fetching data
- **Form submission**: Disabled buttons with loading state
- **Delete operation**: Loading indicator during deletion

## Client Integration

Projects are tightly integrated with the Clients module:

- **Client dropdown**: Select from existing clients when creating/editing
- **Client name display**: Automatically shows client name on project cards
- **Client deletion**: Deleting a client removes all associated projects

<Info>
  Make sure to create clients before creating projects that reference them.
</Info>
