## Client Data Structure

Each client record contains the following information:

```typescript
interface Client {
  id: string;              // Unique identifier
  name: string;            // Client/company name
  contactPerson: string;   // Primary contact name
  email: string;           // Contact email address
  phone: string;           // Contact phone number
}
```

<Info>
  All fields are required when creating or editing a client to ensure complete contact information.
</Info>

## Managing Clients

<Tabs>
  <Tab title="Create Client">
    ### Adding a New Client
    
    <Steps>
      <Step title="Open Client Form">
        Click the **"Nuevo Cliente"** button at the top of the Clients page.
      </Step>
      
      <Step title="Enter Details">
        Fill in all required fields:
        - **Name**: Client or company name
        - **Contact Person**: Primary contact's full name
        - **Email**: Valid email address
        - **Phone**: Contact phone number
      </Step>
      
      <Step title="Save Client">
        Click save to add the client to your portfolio. They'll appear in the client grid immediately.
      </Step>
    </Steps>
    
    <Note>
      New clients can be immediately associated with projects after creation.
    </Note>
  </Tab>
  
  <Tab title="Edit Client">
    ### Updating Client Information
    
    To modify an existing client:
    
    1. Locate the client in the grid
    2. Click the **EDITAR** button on their card
    3. Update any fields in the form
    4. Save changes to update the client record
    
    <Info>
      Editing a client's name will update how they appear in all associated projects.
    </Info>
  </Tab>
  
  <Tab title="Delete Client">
    ### Removing a Client
    
    <Warning>
      Deleting a client will **permanently remove**:
      - The client record
      - All associated projects
      - All project-related data (notes, links, prompts)
      
      This action cannot be undone!
    </Warning>
    
    To delete a client:
    
    1. Click the **ELIMINAR** button on the client card
    2. Review the confirmation dialog
    3. Confirm to permanently delete the client and all associated data
  </Tab>
</Tabs>

## Client Operations

The Clients component handles these API operations:

```typescript
// Fetch all clients
const data = await getClients();

// Add new client
await addClient(clientData);

// Update existing client  
await updateClient({ ...editingClient, ...clientData });

// Delete client
await deleteClient(clientId);
```

<Info>
  All operations refresh the client list automatically to keep data synchronized.
</Info>

## Loading States

The module provides visual feedback during operations:

- **Initial load**: Large centered spinner while fetching clients
- **Form submission**: Disabled buttons prevent duplicate submissions
- **Delete operation**: Loading indicator during deletion

```typescript
const [loading, setLoading] = useState(true);
```

## Confirmation Dialogs

Critical operations require confirmation:

<Warning>
  The delete confirmation dialog explicitly warns about cascading deletions:
  
  _"¿Estás seguro de que quieres eliminar este cliente? Esta acción también eliminará todos sus proyectos asociados y no se puede deshacer."_
</Warning>

This prevents accidental data loss from cascade deletions.

## Integration with Projects

Clients and projects are tightly integrated:

### How They Connect

1. **Project Creation**: Select from existing clients when creating projects
2. **Project Display**: Client name appears on project cards
3. **Cascade Deletion**: Removing a client deletes all associated projects

```typescript
interface Project {
  clientId: string;      // References client by ID
  clientName?: string;   // Cached client name for display
  // ... other fields
}
```

<Info>
  The project's `clientId` field creates the relationship between clients and projects.
</Info>

The module includes error handling for:

- Failed API requests
- Network issues
- Invalid form data
- Deletion conflicts

Errors are logged to the console for debugging:

```typescript

}
```
