## Overview

The GestorPro Dashboard provides a comprehensive overview of your business operations at a glance. It aggregates data from projects, clients, and bots to deliver actionable insights through summary cards, interactive charts, and upcoming deliveries tracking.

<Note>
  The dashboard automatically refreshes bot status data every 30 seconds to keep metrics up to date.
</Note>

const projectStatusData = useMemo(() => {
  const statusCounts = {
    [ProjectStatus.NotStarted]: 0,
    [ProjectStatus.InProgress]: 0,
    [ProjectStatus.Completed]: 0,
    [ProjectStatus.OnHold]: 0,
  };
  projects.forEach(p => {

## Upcoming Deliveries

The dashboard displays the next four upcoming project deadlines to help prioritize work:

**Display Logic:**
1. Filters out completed projects
2. Filters out past deadlines
3. Sorts by end date (earliest first)
4. Shows top 4 upcoming deliveries

**Information Shown:**
- Project name
- Associated client name
- Delivery date (formatted)
- Visual card layout

```typescript Dashboard.tsx:134-147
{projects
  .filter(p => p.status !== ProjectStatus.Completed && new Date(p.endDate) > new Date())
  .sort((a, b) => new Date(a.endDate).getTime() - new Date(b.endDate).getTime())
  .slice(0, 4)
  .map(project => (
    <div key={project.id} className="flex items-center justify-between p-3 bg-gray-50 rounded-lg">
      <div>
        <p className="font-semibold text-gray-800">{project.name}</p>
        <p className="text-sm text-gray-500">{project.clientName}</p>
      </div>
      <span className="font-medium text-indigo-600">{new Date(project.endDate).toLocaleDateString()}</span>
    </div>
  ))
}
```

<Info>
  If no upcoming deliveries exist, the dashboard displays a friendly "No hay entregas pendientes" message.
</Info>

## Data Loading

The dashboard implements a dual-loading strategy for optimal performance:

<Steps>
  <Step title="Initial Load">
    Fetches all data in parallel using `Promise.all` for projects, clients, and bots
  </Step>
  <Step title="Real-time Updates">
    Polls bot status every 30 seconds to keep the "Active Bots" metric current
  </Step>
  <Step title="Loading State">
    Displays an animated spinner during initial data fetch
  </Step>
</Steps>

```typescript Dashboard.tsx:34-56
useEffect(() => {
  const fetchInitialData = async () => {
    setLoading(true);
    try {
      const [projectsData, clientsData, botsData] = await Promise.all([
        getProjects(), 
        getClients(),
        getBots()
      ]);
      setProjects(projectsData);
      setClients(clientsData);
      const activeBots = botsData.filter(bot => 
        bot.status?.toUpperCase() === 'ONLINE' || bot.status?.toUpperCase() === 'ACTIVE'
      ).length;
      setActiveBotsCount(activeBots);
    } catch (error) {
      console.error("Error al cargar datos del dashboard:", error);
    } finally {
      setLoading(false);
    }
  };
  fetchInitialData();
}, []);
```

## Performance Optimizations

The dashboard uses several React performance patterns:

<AccordionGroup>
  <Accordion title="useMemo for Expensive Calculations">
    Total budget and project status data are memoized to prevent unnecessary recalculations
  </Accordion>
  <Accordion title="Parallel Data Fetching">
    All initial data loads simultaneously using `Promise.all` instead of sequential requests
  </Accordion>
  <Accordion title="Selective Polling">
    Only bot data refreshes periodically; projects and clients load once on mount
  </Accordion>
  <Accordion title="Cleanup on Unmount">
    The bot polling interval is cleared when the component unmounts to prevent memory leaks
  </Accordion>
</AccordionGroup>
